Fault Detection, Isolation, And Recovery FDIR / FMEA
CPU_OVERLOAD
Detected by health monitor. Contained by DEGRADED_PAYLOAD mode. Recovered by CLEAR_FAULT + SET_MODE.
BAD_COMMAND_CRC
Detected before command dispatch. Packet is dropped and cannot mutate FSW state.
WATCHDOG_STALL
Detected by heartbeat supervision. Drives watchdog fault path and recovery tests.
SENSOR_TIMEOUT
Detected by stale sensor health input. Covered by scenario tests and safe-mode path.

Engineering Trade Study Why This Architecture
UDP vs TCP
UDP

Chosen because command/telemetry links in embedded test rigs often prioritize low overhead, explicit packet validation, and application-level sequencing.

Pi HIL vs Sim Only
HIL

Pi execution proves the C++ target builds and runs on non-x86 hardware, catching architecture and deployment issues.

Replay vs Live HIL
Both

Replay provides deterministic evidence review. Live HIL execution remains available through the command/telemetry tools and evidence workflow.
