AstraSim-FSW — Flight Software HIL Verification Workstation

Complete
  • C++17
  • Python
  • Raspberry Pi (aarch64)
  • UDP / MAVLink
  • CRC-16

Problem

Flight software requires rigorous verification before deployment, but existing HIL (Hardware-in-the-Loop) setups are often bulky or inaccessible for rapid student iteration.

Approach

Designed a C++17/Python flight-software-in-the-loop framework utilizing UDP command/telemetry links with binary packets. Implemented CRC-16 validation, fault injection, and watchdog logic. Deployed on Raspberry Pi 4B for aarch64 architecture verification.

Result

Achieved 9/9 CTest suites passing, 5/5 YAML scenario runs, and 25/25 Monte Carlo regressions. Successfully validated Pi HIL evidence, proving the system's viability for rapid FSW testing.

Reflection & Lessons Learned

Initial CRC implementation caused noticeable latency on the Pi. Researched and migrated to a CRC-16-CCITT lookup table approach, reducing packet validation overhead by 40%. Next iteration will focus on integrating a real-time clock for stricter timing constraints.

GHOST — GPS-Denied Hardware Occlusion-Survivable Tracker

Active
  • EKF / ESKF
  • Raspberry Pi 4B
  • PX4 SITL
  • IMU / Vision Fusion

Problem

Standard GPS navigation fails in urban canyons or occluded environments. A robust system is needed that can survive signal loss and maintain state estimation using alternative sensors.

Approach

Developing a 9-DOF Error-State Kalman Filter (ESKF) fusing IMU data with vision pipeline inputs. Running on Raspberry Pi 4B with an IMX296 camera and ICM-42688-P IMU. Integrating with PX4 SITL via UDP/MAVLink for real-time testing.

Result

Filter state currently nominal. Pipeline completion at 55%. Successfully established UDP MAVLink connection to PX4 SITL for data injection.

Reflection & Lessons Learned

Initial sensor synchronization drift caused the ESKF to diverge during rapid motion. Solved by implementing hardware-level interrupt triggering for the IMU to align timestamps with camera frame captures. Currently optimizing the vision pipeline to run faster than 30Hz on the Pi.
